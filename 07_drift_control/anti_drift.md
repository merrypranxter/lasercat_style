# Anti-Drift

Drift is the enemy of `laser_cat_style`.

This file defines what drift is, why it happens, and how to prevent it.

---

# What is drift?

Drift is when an image or output moves away from the core identity of `laser_cat_style` toward:
- generic space cat art
- generic trippy cat art
- generic meme collage
- polished poster design
- serious fantasy art
- kawaii cat illustration
- vaporwave aesthetics
- random internet nonsense

Drift is not about quality.
It is about identity.

A high-quality image can drift.
A low-quality image can be core.

---

# Why drift happens

## 1. Over-polishing
Too much refinement removes the crust that makes the style authentic.

## 2. Under-escalation
Too little visual intensity makes the image boring.

## 3. Wrong motif families
Using objects or effects that do not belong to the style.

## 4. Loss of cat centrality
The cat becomes just another element.

## 5. Loss of collage logic
The image becomes seamless realism.

## 6. Wrong tone
The image becomes too serious, too cute, or too cool.

## 7. Generic composition
The image uses no recognizable style structure.

---

# Anti-drift principles

## 1. Preserve cat centrality
The cat must remain the main subject, power source, or joke driver.

## 2. Maintain collage visibility
The image should feel assembled, not photographed.

## 3. Use correct motif families
Stick to lasers, rainbows, food, glam, UFOs, household objects, co-stars.

## 4. Keep the tone absurd
The image should feel stupid, campy, colorful, and internet-native.

## 5. Use correct composition modes
Sticker scatter or single-gag hero.

## 6. Preserve correct badness
Some roughness, glow, or artifact is part of the style.

## 7. Maintain the deity/idiot balance
The cat should be powerful and stupid simultaneously.

---

# Anti-drift checklist

Before finalizing an image, ask:

## Cat
- [ ] Is the cat central?
- [ ] Is the cat readable?
- [ ] Does the cat have expression?
- [ ] Is the cat powerful and stupid?

## Collage
- [ ] Does the image feel assembled?
- [ ] Is there visible cutout or compositing?
- [ ] Is there some "wrongness" preserved?

## Motifs
- [ ] Are the objects from correct families?
- [ ] Is there at least one style-native motif?

## Composition
- [ ] Is the mode recognizable?
- [ ] Is there clear hierarchy?

## Tone
- [ ] Is the image absurd?
- [ ] Is it campy or tacky?
- [ ] Is it internet-native?

## Escalation
- [ ] Is the image pushed beyond ordinary?
- [ ] Is there visual intensity?

# Common drift types

## Type 1: Polished drift
The image becomes too refined, too clean, too designed.

Fix:
- add roughness
- add glow
- add artifact
- reduce realism
- increase tackiness

## Type 2: Generic drift
The image becomes "cat in space" with no specific style logic.

Fix:
- add specific motifs
- add absurd context
- add object nonsense
- strengthen composition mode

## Type 3: Serious drift
The image becomes genuine fantasy or sci-fi art.

Fix:
- add cat idiocy
- add absurd context
- add tacky elements
- reduce cinematic drama

## Type 4: Cute drift
The image becomes kawaii without absurdity.

Fix:
- add stupid premise
- add cosmic escalation
- add object nonsense
- reduce cuteness discipline

## Type 5: Cool drift
The image becomes vaporwave or design-forward.

Fix:
- add goblin energy
- add food or household nonsense
- reduce graphic restraint
- increase tackiness

## Type 6: Random drift
The image has no coherent joke or logic.

Fix:
- establish clear premise
- strengthen cat centrality
- add recognizable motif
- improve composition hierarchy

---

# Anti-drift strategies

## Strategy 1: Reference core examples
Compare outputs to known good examples.

## Strategy 2: Use the checklist
Run every image through the anti-drift checklist.

## Strategy 3: Strengthen weak poles
If the image is too serious, add idiocy.
If the image is too random, add structure.

## Strategy 4: Add style-native motifs
Introduce lasers, rainbows, food, glam, or UFOs.

## Strategy 5: Adjust composition mode
Switch between sticker scatter and single-gag hero.

## Strategy 6: Modify artifact level
Add or reduce glow, roughness, or crust.

---

# Procedural anti-drift

For automated systems:

## Constraints
- Enforce cat centrality rules
- Limit motif families to style-native options
- Require recognizable composition mode
- Maintain tone constraints

## Diagnostics
- Run drift detection algorithms
- Compare to style embeddings
- Flag potential drift types

## Repair
- Apply automatic corrections
- Suggest style-native alternatives
- Re-generate with stronger constraints

---

# Bottom line

Drift is the main threat to `laser_cat_style` identity.

Anti-drift requires:
- awareness of drift types
- use of the checklist
- application of correction strategies
- procedural constraints

The goal is not perfection.
The goal is **identity preservation**.

A slightly rough image that is core `laser_cat_style` is better than a polished image that has drifted.
